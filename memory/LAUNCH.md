## 📋 Play Console submission flow

### Phase 1 — One-time setup (skip if already done)
1. ✅ Create Play Console developer account ($25 one-time): https://play.google.com/console/signup
2. ✅ Accept Developer Distribution Agreement
3. ✅ Set up payment profile *(only needed for paid apps / IAP — not for free)*

### Phase 2 — Create app entry
1. Play Console → **Create app**
2. Name: `RouTeD: AI Route Optimizer`
3. Default language: English (Australia)
4. App or game: **App**
5. Free or paid: **Free** *(IAP for Pro tier handled in-app)*
6. Accept declarations (developer program policies, US export laws)
7. Click **Create app**

### Phase 3 — Fill the dashboard tasks (top of app dashboard)
Play Console shows a checklist at the top. Hit them in order:

| Task | What to do |
|---|---|
| **App access** | "All functionality available without restrictions" OR provide reviewer creds (`routedreviewer@gmail.com` + the passcode from `test_credentials.md`) |
| **Ads** | "No, my app does not contain ads" |
| **Content rating** | Complete questionnaire — RouTeD is **Everyone / PEGI 3** (no violence, no chat, no UGC) |
| **Target audience** | Age 18+ *(driver tool — not for kids)* |
| **News app** | "No, my app is not a news app" |
| **COVID-19 contact tracing** | No |
| **Data safety** | Declare: GPS location collected (used for navigation, shared with no one, encrypted in transit + at rest); device ID collected (used for analytics, deletable); user can request deletion. **Link your privacy policy URL.** |
| **Government app** | No |
| **Financial features** | No |
| **Health** | No |
| **Store listing** | Paste copy from this doc; upload icon, feature graphic, all 8 screenshots |
| **App category** | Maps & Navigation |
| **Store settings — Contact details** | Your support email + website URL |
| **Main store listing** | Save |

### Phase 4 — Internal testing track FIRST (do NOT submit to Production directly)
1. Play Console → **Testing → Internal testing**
2. **Create new release**
3. Upload your `.aab` file
4. Release name: `1.0.0-internal-1` *(matches your `app.json` version)*
5. Release notes (paste as-is):
   ```
   First release — RouTeD 1.0.0
   - 14-solver optimization pipeline
   - 60fps driving cockpit
   - ML service-time + building-side learning
   - CSV/XLSX import with async geocoding
   - Route history + resume
   ```
6. **Save → Review release → Start rollout to Internal testing**
7. Add yourself + 1-2 testers as testers (Internal testing → Testers → Create email list)
8. Install via the **opt-in link** Play Console gives you — test the AAB end-to-end

### Phase 5 — Promote to Production
Once Internal testing is solid:

1. Play Console → **Production → Create new release**
2. **Use existing AAB → pick the one from Internal testing**
3. Same release notes
4. **Countries:** Start with **Australia** only *(easier to triage Play Console warnings; expand later via "Add countries")*
5. **Rollout %:** Start at **20%** *(catches issues before all users hit them)*
6. **Save → Review release → Start rollout to Production**

### Phase 6 — Wait for Google review
- First-ever submission: **3-7 days** typical
- Subsequent updates: **1-2 days** typical
- Watch the dashboard for status changes ("In review" → "Published")

### Phase 7 — If rejected
Common rejection reasons + fixes:

| Reason | Fix |
|---|---|
| Missing permission justification | Background location: add the foreground-service rationale (already in app.json, but copy to Play Console "Data Safety" form too) |
| Privacy policy link broken | Verify `https://floating-map-ui.emergent.host/privacy` returns HTTP 200 (already checked — it does) |
| Target API level too low | We target API 35, current Play Store requirement is 34+, you're fine |
| Crashes on test device | Check Play Console → Quality → Android vitals for the crash stack; ship a hotfix AAB |
| Misleading screenshots | Make sure screenshots accurately show in-app screens (they're real screenshots from `scripts/generate_playstore_screenshots.py`, so you're fine) |
